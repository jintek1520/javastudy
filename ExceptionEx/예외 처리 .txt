

5Ways Exception Handling

1. try ~ catch : 예외가 발생했어도 예외가 없는 것과 같은 효과
-try : 예외가 발생할 수 있는 코드 -catch : 예외가 발생할 때 실행할 코드

2.throws : 예외가 발생했을 때 , 해당 메서드를 호출 한 곳에서 예외처리를 하도록 예외를 던져줌.

3.예외가 발생하든 그렇지 않든 항상 실행해야 하는 코드는 finally에 구현. return 보다 더 강력함.

4.throw : 문법적으로 예외는 아니지만 논리적인 예외를 발생시키는 기능을 위해 쓴다.

5.사용자 정의 예외 클래스 : 가능하면 Exception을 상속하도록 하여, throws을 날리도록 구현하라 그러면 호출하는 메서드에서 try~catch 작업이 가능하니까.1.
